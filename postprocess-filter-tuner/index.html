2. Filters

Which knobs, and why only these?

These mirror sleap-nn's post-inference FilterConfig and run on the already-predicted scores/coords, so they reproduce inference exactly. Inference-time knobs (peak_threshold, integral_refinement, center_nms_kernel) act on confidence maps and cannot be reproduced from a .slp — they're intentionally omitted.

Matching & scoring parameters
sleap-nn match_threshold (keypoint models): min OKS for a pred to match a GT (greedy, by instance score). 0 = default.
Max pixel distance for a centroid to match a GT (Hungarian). Drives the centroid-detection view and is the primary matcher for centroid-only models. Default 50.
A keypoint within this distance of GT is "correct". Drives PCK and the good-vs-error split.
